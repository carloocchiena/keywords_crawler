What is this?

This is a keyword finder.

I created this web app myself from scratch, starting from a Python script I made for my own use.

The Problem: To quickly understand what a site and its articles are about without delving into reading the articles themselves, and having an easily exportable keyword trail, presented me with two problems:
- navigating the whole site
- understanding the definition of keywords

The Solution: To solve this, I first created a crawler that iteratively scans the whole site for relevant urls, through a search & purge process.
Then through a data cleaning&sorting algorithm it returns the statistically relevant keywords.
For each url we then have the keyword and its relevance score.

Since I liked the raw script in Python quite well, I exported it to a web version. I wanted a "full stack Python" approach, so I switched to Flask (after several experiments in Django, which were overkill for the simplicity of the app).

The goal is to make it a tool that can be used by third parties without the need to master Python.

This first release is still quite rough.

Known Issues

  • Algo performances (scrapers rely on website speed, and also the functions are not yet tweaked and optimized)
  • Websites with no nidified urls or no urls
  • Websites that doesn't allow crawlers
  • Weird-coded websites (the scraper relies on a canonical DOM structure)
